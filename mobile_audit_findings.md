## Key findings

### Global
- Mobile site is functional and responsive, but spacing feels compressed versus desktop.
- Several sections use large desktop paddings and card densities that create long pages without enough visual hierarchy.
- Footer is especially tall on mobile and pushes important content far down.

### Homepage
- Hero is readable, but headline and paragraph stack tightly and feel oversized relative to available width.
- CTA buttons are large and usable, but surrounding spacing could be tightened for a cleaner first screen.
- Mid-page cards and sections appear dense; typography rhythm needs improvement.

### Guide pages
- Guide hero metadata wraps awkwardly and CTA block sits too close to heading content.
- Long text sections need slightly better mobile spacing and content width handling.
- Breadcrumb and title relationship needs cleaner spacing on small screens.

### Schools page
- Filters stack correctly but occupy a lot of vertical space.
- School cards appear very dense and text-heavy on mobile.
- Testimonial and metadata blocks likely need better spacing and hierarchy.

### About page
- Founder card works, but section spacing can be tightened.
- Footer remains extremely tall after page content.

### Guides index
- Card list is readable but dense; spacing between cards and category sections can improve.
- Footer length again hurts overall mobile polish.

### Jobs / Stories
- Hero sections are acceptable, but cards likely need improved padding and vertical rhythm.

## Priority fixes
1. Improve global mobile spacing scale and container padding.
2. Optimise nav, hero sections, and CTA/button spacing.
3. Improve GuideLayout mobile typography, metadata wrapping, and related blocks.
4. Reduce density in schools, jobs, stories, and guides index cards.
5. Simplify footer layout on mobile to reduce excessive page length.
6. Rebuild and re-check screenshots after changes.
